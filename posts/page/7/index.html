如何利用本地工作副本建立远程GIT库

Mar 2nd, 2011 | Comments

1, 首先备份 .git/config
打开 config , 只保留如下部分[core]
repositoryformatversion = 0
filemode = true
bare = false
logallrefupdates = true

2, 回到.git同级目录执行
git remote add origin ssh://user@123.321.123.321:88888/home/user/git/neten.git

3, 再次打开.git/config
发现会变成这个样子了:
[core]
repositoryformatversion = 0
filemode = true
bare = false
logallrefupdates = true

[remote “origin”]
url = ssh://user@123.321.123.321:88888/home/user/git/neten.git
fetch = +refs/heads/:refs/remotes/origin/
我们需要添加一个单元, 最后是这样子:[core]
repositoryformatversion = 0
filemode = true
bare = false
logallrefupdates = true

[remote “origin”]
url = ssh://user@123.321.123.321:88888/home/user/git/neten.git
fetch = +refs/heads/:refs/remotes/origin/
[branch “master”]
remote = origin
merge = refs/heads/master
  
4, 登录服务器123.321.123.321, 建立远程git库
$ cd /home/user/git
$ mkdir neten.git
$ cd neten.git
$ git init —bare

5, 回到本地执行
$ git push origin master

Ubuntu下改变mysql数据库路径

Feb 27th, 2011 | Comments

1, 停止MYSQL服务
/etc/init.d/mysql stop

2, 把数据库文件移动到你指定的位置
mv /var/lib/mysql/ /home/neten/DB/

3, 设置好目录权限, 假设测试数据库为test
cd /home/neten/DB/
chown mysql:mysql ./mysql -R
chmod 700 /home/neten/DB/mysql/test/
chmod 660 /home/neten/DB/mysql/test/*

4, 修改配置文件
sudo vi /etc/my.cnf
查找 /var/lib/mysql, 替换成 /home/neten/DB/mysql

5, 修改mysqld可以对哪些路径进行操作
sudo vi /etc/apparmor.d/usr.sbin.mysqld
查找 /var/lib/mysql, 替换成 /home/neten/DB/mysql
/etc/init.d/apparmor restart

6, 启动MYSQL服务
/etc/init.d/mysql start

把域名从Google Apps转到Godaddy

Nov 1st, 2010 | Comments

以前在Google Apps注册了域名, 现在想转到Goddy
http://www.google.com/support/a/bin/topic.py?topic=14579

1, 首先登录Google Apps, 进而登录Godaddy的面板
2, 点击域名, Unlock之
3, 在 Authorization code 右边点击 Send by email
4, 在Godaddy注册要转移的域名, 注意是Transfer Domain
5, 在所有域名的管理面板点击Pending Transfers, 选中域名, 在右上角点击Authorization
6, 接收Email, 现在已经得到了
Transaction ID: 123456789
Security Code: NABC12345
Authorization code 是第三步得到的
7, 唯一的障碍就是Cancel Private Registration
http://help.securepaynet.net/article/1625?prog_id=412956

    1) Log in to your account at www.DomainsByProxy.com.
      登录信息应该和Google Apps里用来登录Godaddy的帐户, 如果不行, 就点击Forgot your login information
    2) 在每行域名列表的最后面点击Cancel private registration
8, 回到Godaddy自己的帐号下, 在所有域名的管理面板点击Pending Transfers, 选中域名, 在右上角再次点击Authorization, 输入相应信息, 搞定.

如果是enom的域名, 上面的1,2,3步可省略, 直接给下面的Email发邮件就好了
GoogleClients@enom.com
记得把Google Apps提供的用于登录的域名和密码写在email里面就可以了.

POKA-YOKE [ZT]

Jul 27th, 2009 | Comments

日本丰田汽车公司的一位名叫Shigeo Shingo的工程师，利用一种被称作Poka-yoke(Pronounced POH-kah YOH-kay)的设备创立了一套质量管理方法（Poka-kah 的日文意思是防范差错）。

所谓Poka-yoke，它可以是任何一种机械装置，它能够防止人为错误的发生或者是让人一眼就能够找到出现错误的位置。也就是说，Poka-yoke装置的用途包括两个方面，一个是杜绝产生特定产品缺陷的原因，另一个是通过廉价的手段对生产产品进行逐一的检查，以确定其是否合格。这种检查的操作员在执行组装的过程中完成的，它们对操作员应该是透明的。也就是说，只要Poka-yoke的条件不满足，则操作就不能继续下去。

POKA-YOKE防呆法/愚巧法/防错法,更多是一种理念,而不是工具.因为它让你从预防的角度去想尽可能让笨蛋都不会犯错. 这很难啊!尽力去做就好了.

以前冲模工序老有人冲伤,原因是脚踏式的,手脚不协调而致.后来有人发明了手动按钮, 冲伤就减少了,因为手比脚灵活多了.但是还是有人冲伤,原因是一手按按钮,一手拿冲好的产品还是会有危险,所以就发明了双制动按钮.很多人都知道这是个典型的POKAYOKE例子,但是并非到此结束了.

今年前我们公司的冲模工序就发生工伤,虽然是双制动按钮,但是工人为了加快速度,就用肘或膝盖顶住一个按钮,腾出手来取放产品.
原来的防呆就不防呆了.怎么办呢?

后来工程师们就取了一个当时是认为防呆的办法.把原来的双制动按钮改成内陷式的.你说好吗?
